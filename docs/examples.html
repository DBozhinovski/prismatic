Command line

./prismatic -h

Usage: prismatic [options]

Options:

-h, --help                      output usage information
-d, --do [task]                 The task to do [mp4]
-i, --input [input]             Input path
-a --audio_codec [audio_codec]  Audio codec
-v --video_codec [video_codec]   Video codec
-o, --output [output]           Output path
-r, --resolution [resolution]   Resolution
-t, --time [time]               Time

NOTE: The CLI utility part of prismatic was created for testing purposes only. If you need a straight-to-command-line utility, use FFMPEG directly instead.

Examples

Convert to mp4 / h264 / aac
./prismatic -d mp4 -i {input_path} -o {output_path}
  • -d mp4 is optional, since it’s default
Convert to webm / libvpx / libtheora
./prismatic -d webm -i {input_path} -o {output_path}
Get metadata
./prismatic -d meta -i {input_path}
Get a thumbnail
./prismatic -d thumbnail -i {input_path} -t {time} -r {resolution} -o {output_path}
  • time is in seconds and defaults to 120
  • resolution defaults to 1024x768
Manual mode
./prismatic -d manual {mix_and_match_available_options}
  • Warning: the manual mode usually doesn’t work. As mentioned, it was used for testing purposes only.

Library

The whole library is consisted of 3 classes - Plumbing, Porcelain and Formatter. (Names obviously inspired by git :) )

Plumbing

Low level wrapper for ffmpeg. When instantiated via the @get method, it return an object with two functions - instance and meta.

  • instance spawns a child_process with ffmpeg, binds a Q promise to it and returns that promise.
    • It reports data on stderr output (since ffmpeg logs output to stderr) and resolves the promise when the stream closes.
  • meta returns metadata for the given path (via exec and ffprobe)

Porcelain

“Higher” lever wrapper over plumbing, that formats input via the Formatter class. Methods include:

  • @getPreviewImage - test method, used for the CLI utility. Gets thumbnails.
  • @getWebM - test method, transcodes a video to webm.
  • @getMp4 - test method, transcodes a video to mp4.
  • @manual - the method used by the previous 3. It uses the instance method from Plumbing to spawn an instance, and passes input previously processed by the Formatter class, in order to ensure that the input is sane.
  • @info - returns metadata. Redundant, and scheduled for change in the next version.

Formatter

As the name implies, a formatting utility class. Methods include:

  • @input - Formats input from various forms into an array, suitable for child_process.spawn
  • @formatFromStdOut - parses stdout (in this case stderr) data from ffmpeg, and returns a json from the output. The second argument is a callback of sorts, or the function that the formatted output is passed to.
  • @streamToStdOut - Logs data to stdout with all the changing attributes obtained from ffmpeg.
  • @returnProgress - formats from stdout, calculates time and does some sanity checks to avoid NaN.
  • @printToStdOut - raw output console.log, used for debugging
  • @calculatePercentage - calculates progress via given current time and target time.
